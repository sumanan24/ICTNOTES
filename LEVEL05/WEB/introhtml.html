What is HTML?

HTML stands for HyperText Markup Language

  • Standard markup language for creating web pages
  • Describes the structure of a web page
  • Consists of a series of elements
  • Elements tell the browser how to display content
  • Not a programming language - it's a markup language
4/100

History of HTML

  • 1991: HTML 1.0 - Tim Berners-Lee created
  • 1995: HTML 2.0 - First standard version
  • 1997: HTML 3.2 - W3C Recommendation
  • 1999: HTML 4.01 - Widely adopted
  • 2014: HTML5 - Current standard
  • Living Standard: Continuously updated by WHATWG
5/100

HTML Document Structure

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>Page Title</title> </head> <body> <!-- Content goes here --> </body> </html>

Every HTML document follows this basic structure.

6/100

DOCTYPE Declaration

<!DOCTYPE html>
  • Must be the first line in an HTML document
  • Tells the browser which version of HTML the page is written in
  • HTML5 uses the simplified declaration shown above
  • Not case-sensitive but conventionally written in uppercase
  • Ensures the browser renders the page in standards mode
7/100

The <html> Element

<html lang="en"></html>
  • Root element of an HTML page
  • Contains all other HTML elements
  • lang attribute specifies the language of the document
  • Important for search engines and accessibility
  • Example: lang="en" for English, lang="es" for Spanish

The <head> Element

  • Not displayed on the page
  • Includes: title, character set, styles, scripts, and other meta information
  • Placed between <html> and <body> tags
9/100

The <body> Element

<body></body>

  • Includes: headings, paragraphs, images, links, tables, lists, etc.
  • Only one <body> element per document

Block vs Inline Elements

Block-level Elements:

  • Start on a new line
  • Take up full width available
  • Examples: <div>, <p>, <h1>, <ul>

Inline Elements:

  • Don't start on a new line
  • Only take up necessary width
  • Examples: <span>, <a>, <strong>, <img>

What is CSS?

CSS stands for Cascading Style Sheets

  • Describes how HTML elements are displayed
  • Controls layout, colors, fonts, spacing
  • Separates presentation from content
  • One CSS file can style multiple pages
  • External stylesheets save work and load faster
62/100

CSS Syntax

selector { property: value; property: value; }
h1 {
  color: blue;
  font-size: 24px;
}
  • Selector: Targets HTML element
  • Property: Aspect to style
  • Value: Setting for property
63/100

Three Ways to Insert CSS

1. Inline CSS:

<p style="color: red;">Text</p>

2. Internal CSS:

<style>p { color: red; }</style>

3. External CSS:

<link rel="stylesheet" href="styles.css">

External is the best practice!

CSS Specificity

When multiple rules target the same element, specificity determines which applies:

  • Inline styles: 1000 points
  • IDs: 100 points
  • Classes, attributes, pseudo-classes: 10 points
  • Elements, pseudo-elements: 1 point
#header .nav a = 111 points

Cascade and Inheritance

  • Multiple styles can apply to same element
  • Last rule wins if equal specificity
  • !important overrides (use sparingly)

Inheritance:

  • Some properties inherit from parent elements
  • Color, font properties typically inherit
  • Layout properties typically don't
