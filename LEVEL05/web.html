1Web Contents Development 1 (HTML)

Understand and Explain the Outline of HTML

HTML (HyperText Markup Language) is the standard markup language used to create web pages. It provides the structure and content of a webpage, defining elements like headings, paragraphs, links, images, and more.

HTML Document Structure: Every HTML document follows a specific structure that includes:

  • DOCTYPE Declaration: Tells the browser which version of HTML is being used
  • HTML Element: The root element that contains all other elements
  • Head Section: Contains metadata, title, and links to external resources
  • Body Section: Contains the visible content of the webpage
<!-- Basic HTML5 Document Structure --> <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>My First Web Page</title> </head> <body> <h1>Welcome to HTML</h1> <p>This is a paragraph of text.</p> </body> </html>
Key HTML Concepts:
  • Elements: Building blocks of HTML (e.g., <p>, <div>, <h1>)
  • Tags: Markers that define elements (opening <tag> and closing </tag>)
  • Attributes: Additional information about elements (e.g., id, class, src, href)
  • Nesting: Placing elements inside other elements
  • Semantic HTML: Using meaningful tags that describe content purpose

Identify HTML Basic Tags

HTML provides numerous tags for structuring content. Understanding these basic tags is fundamental to web development.

Tag Description Example
<h1> to <h6> Heading tags (h1 is largest, h6 is smallest) <h1>Main Title</h1>
<p> Paragraph text <p>This is a paragraph.</p>
<a> Anchor (hyperlink) <a href="page.html">Link</a>
<img> Image (self-closing) <img src="image.jpg" alt="Description">

<input> Form input field <input type="text" name="username">
<button> Clickable button <button type="submit">Submit</button>

1.5Web Styling (CSS)

Understand CSS Syntax, Selectors, Box Model, Flexbox/Grid Layouts

CSS (Cascading Style Sheets) is used to style and layout web pages. It controls colors, fonts, spacing, positioning, and responsive design.

CSS Syntax:

/* CSS Syntax: selector { property: value; } */ selector { property: value; property: value; } /* Example */ h1 { color: blue; font-size: 2em; margin: 10px; }

CSS Selectors:

Selector Type Syntax Example
Element element p { color: red; }
Class .classname .highlight { background: yellow; }
ID #idname #header { height: 100px; }
Descendant parent child div p { margin: 10px; }
Child parent > child ul > li { list-style: none; }
Attribute [attribute] input[type="text"] { border: 1px solid; }

Box Model:

The CSS box model describes how elements are rendered with content, padding, border, and margin.

.box { /* Content area */ width: 200px; height: 100px; /* Padding - space inside border */ padding: 20px; /* Border - around padding */ border: 2px solid black; /* Margin - space outside border */ margin: 10px; /* Box-sizing controls how width/height are calculated */ box-sizing: border-box; /* includes padding and border */ }

Flexbox Layout:

/* Flexbox Container */ .container { display: flex; flex-direction: row; /* or column */ justify-content: center; /* main axis alignment */ align-items: center; /* cross axis alignment */ flex-wrap: wrap; gap: 20px; } /* Flexbox Items */ .item { flex: 1; /* grow, shrink, basis */ flex-grow: 1; flex-shrink: 1; flex-basis: 200px; }

CSS Grid Layout:

/* Grid Container */ .grid-container { display: grid; grid-template-columns: repeat(3, 1fr); /* 3 equal columns */ grid-template-rows: auto; gap: 20px; grid-template-areas: "header header header" "sidebar main aside" "footer footer footer"; } /* Grid Items */ .header { grid-area: header; } .sidebar { grid-area: sidebar; } .main { grid-area: main; }

Apply Responsive Design Principles

/* Media Queries for Responsive Design */ @media screen and (max-width: 768px) { .container { flex-direction: column; padding: 10px; } } /* Mobile-first approach */ .responsive { width: 100%; max-width: 1200px; margin: 0 auto; padding: 1rem; } /* Fluid typography */ h1 { font-size: clamp(1.5rem, 5vw, 3rem); }

2Web Contents Development 2 (XML)

Understand and Explain the Outline of XML

XML (eXtensible Markup Language) is a markup language designed to store and transport data. Unlike HTML, XML focuses on data structure rather than presentation.

Key XML Characteristics:
  • Self-descriptive: Tags describe the data they contain
  • Platform independent: Works across different systems
  • Extensible: You can define your own tags
  • Structured: Hierarchical data organization
  • Case-sensitive: Tags must match exactly

Basic XML Structure:

<?xml version="1.0" encoding="UTF-8"?> <root> <element attribute="value"> Content here </element> <child> <nested>Nested content</nested> </child> </root>

Perform XML Document Creation

Creating well-formed XML documents requires following specific rules and structure.

XML Document Example:

<?xml version="1.0" encoding="UTF-8"?> <students> <student id="101"> <name>John Doe</name> <age>20</age> <course>Computer Science</course> <grades> <subject name="Math">85</subject> <subject name="English">90</subject> </grades> </student> <student id="102"> <name>Jane Smith</name> <age>21</age> <course>Information Technology</course> </student> </students>
XML Rules:
  • Must have a root element
  • All tags must be closed
  • Tags are case-sensitive
  • Attributes must be quoted
  • Proper nesting required

Perform XSLT Programming

XSLT (eXtensible Stylesheet Language Transformations) is used to transform XML documents into other formats like HTML, PDF, or other XML structures.

XSLT Example - Transforming XML to HTML:

<?xml version="1.0" encoding="UTF-8"?> <xsl:stylesheet version="1.0" xmlns:xsl="http://www.w3.org/1999/XSL/Transform"> <xsl:template match="/"> <html> <head> <title>Students List</title> </head> <body> <h1>Student Information</h1> <table border="1"> <tr> <th>ID</th> <th>Name</th> <th>Age</th> <th>Course</th> </tr> <xsl:for-each select="students/student"> <tr> <td><xsl:value-of select="@id"/></td> <td><xsl:value-of select="name"/></td> <td><xsl:value-of select="age"/></td> <td><xsl:value-of select="course"/></td> </tr> </xsl:for-each> </table> </body> </html> </xsl:template> </xsl:stylesheet>

XSLT Key Elements:

Element Purpose Example
<xsl:template> Defines transformation rules <xsl:template match="/">
<xsl:value-of> Extracts value from XML <xsl:value-of select="name"/>
<xsl:for-each> Loops through elements <xsl:for-each select="student">
<xsl:if> Conditional processing <xsl:if test="age > 20">
<xsl:choose> Multiple conditions <xsl:choose><xsl:when></xsl:choose>

<?php require_once 'config.php'; // Using MySQLi if ($_SERVER['REQUEST_METHOD'] == 'POST') { $name = mysqli_real_escape_string(getConnection(), $_POST['name']); $email = mysqli_real_escape_string(getConnection(), $_POST['email']); $age = intval($_POST['age']); $conn = getConnection(); $sql = "INSERT INTO users (name, email, age) VALUES (?, ?, ?)"; $stmt = $conn->prepare($sql); $stmt->bind_param("ssi", $name, $email, $age); if ($stmt->execute()) { $message = "User created successfully!"; $messageType = "success"; } else { $message = "Error: " . $stmt->error; $messageType = "error"; } $stmt->close(); $conn->close(); } // Using PDO (More secure) if ($_SERVER['REQUEST_METHOD'] == 'POST') { $pdo = getPDOConnection(); $sql = "INSERT INTO users (name, email, age) VALUES (:name, :email, :age)"; $stmt = $pdo->prepare($sql); $stmt->bindParam(':name', $_POST['name'], PDO::PARAM_STR); $stmt->bindParam(':email', $_POST['email'], PDO::PARAM_STR); $stmt->bindParam(':age', $_POST['age'], PDO::PARAM_INT); if ($stmt->execute()) { $message = "User created successfully!"; } } ?>

PHP MVC Pattern

The MVC (Model-View-Controller) pattern separates application logic into three interconnected components for better organization and maintainability.

PHP MVC Components:
  • Model: Handles data logic and database operations
  • View: Displays data to users (HTML/PHP templates)
  • Controller: Processes requests and coordinates Model and View

MVC Directory Structure:

Project Structure: my-mvc-app/ index.php // Entry point .htaccess // URL rewriting config/ database.php // Database configuration models/ UserModel.php // Data logic views/ users/ index.php // List view create.php // Create form edit.php // Edit form layouts/ main.php // Main layout controllers/ UserController.php // Request handling core/ Router.php // Routing logic Controller.php // Base controller Model.php // Base model

Core Router (core/Router.php):

<?php class Router { private $routes = []; public function addRoute($method, $path, $controller, $action) { $this->routes[] = [ 'method' => $method, 'path' => $path, 'controller' => $controller, 'action' => $action ]; } public function dispatch() { $method = $_SERVER['REQUEST_METHOD']; $path = parse_url($_SERVER['REQUEST_URI'], PHP_URL_PATH); foreach ($this->routes as $route) { if ($route['method'] == $method && $route['path'] == $path) { $controller = new $route['controller'](); $action = $route['action']; $controller->$action(); return; } } http_response_code(404); echo "404 - Page Not Found"; } } ?>

Module 8: Role of a Web Server - Introduction to Internet and Web Publishing (DNS)

Web servers are software that deliver web content to clients over HTTP/HTTPS protocols.

Web Server Functions:
  • Receive HTTP requests from clients
  • Process requests and execute server-side code
  • Return HTTP responses with web content
  • Handle static and dynamic content
  • Manage sessions and security

DNS (Domain Name System):

DNS translates human-readable domain names (e.g., www.example.com) into IP addresses (e.g., 192.168.1.1).

DNS Resolution Process: 1. User types "www.example.com" in browser 2. Browser queries DNS server 3. DNS server returns IP address (e.g., 192.168.1.1) 4. Browser connects to web server at that IP 5. Web server returns requested content Common DNS Record Types: - A Record: Maps domain to IPv4 address - AAAA Record: Maps domain to IPv6 address - CNAME Record: Maps domain alias to another domain - MX Record: Mail exchange server - TXT Record: Text information
