Web Site Design and Hosting

01 Introduction to Web Design (03 Hours)

Learning focus: Why web design matters, web technologies, UX & UI, tools, and current best practices.

1.1 Importance of Web Design

Web design is the process of planning, creating, and maintaining the look, structure, and usability of websites. Good design builds trust and helps users find what they need quickly.

Websites play a key role in business (e.g. online shops, company info, booking) and education (e.g. LMS, course pages, school sites). They support branding and online presence so that people can find and recognise an organisation. The first impression of a site (layout, colours, clarity) affects credibility—a messy or slow site can make users leave.

  • Role of websites in business & education
  • Branding and online presence
  • First impression & credibility

1.2 Overview of Web Technologies

Front-end is what the user sees and uses in the browser (HTML, CSS, JavaScript). Back-end runs on the server (databases, APIs, server logic). A static site shows fixed content; a dynamic site changes content (e.g. from a database or user input).

Type Description
Static Pre-written HTML pages; same for every user
Dynamic Content generated or changed (e.g. login, search, CMS)
  • Front-end vs Back-end
  • Static vs Dynamic websites
  • Basic web architecture (browser ↔ server ↔ database)

1.3 Understanding UX & UI

User Experience (UX) is how a user feels when using a product—ease of use, speed, and whether goals are met. User Interface (UI) is the visual part: buttons, menus, colours, typography. UX is the overall experience; UI is the look and controls. Both must work together for a good website.

  • What is User Experience (UX)?
  • What is User Interface (UI)?
  • UX vs UI differences

1.4 Tools for Web Design

Web authoring tools (e.g. Adobe Dreamweaver, VS Code) help you write and edit HTML, CSS, and JavaScript. Content Management Systems (CMS) like WordPress and Joomla let you build and manage sites with less code—templates, plugins, and an admin panel.

  • Web Authoring Tools (Dreamweaver, VS Code)
  • Content Management Systems (WordPress, Joomla)

1.5 Web Design Trends & Best Practices

Responsive design means the layout adapts to different screen sizes. Minimal layouts reduce clutter and focus on content. Accessibility ensures people with disabilities can use the site (e.g. alt text, keyboard support). Mobile-first means designing for small screens first, then scaling up.

  • Responsive design
  • Minimal layouts
  • Accessibility
  • Mobile-first approach

02 Wireframing and Prototyping (06 Hours)

Learning focus: Plan layout with wireframes, then build clickable prototypes to test user flow before coding.

2.1 Introduction to Wireframing

A wireframe is a simple, often black-and-white sketch of a web page. It shows where elements (header, menu, content, footer) go—without colours or final design. It helps agree structure before spending time on visuals.

Low-fidelity (lo-fi) wireframes are rough (e.g. boxes and lines on paper or simple shapes). High-fidelity (hi-fi) wireframes are more detailed, closer to the final layout, sometimes with real text and spacing.

2.2 Introduction to Prototyping

A prototype is a clickable model of the site. Buttons and links work so you can test user flow (e.g. “Click Login → Enter details → See dashboard”). Interactive prototypes help find usability issues before development.

03 Fundamentals of Web Technologies (03 Hours)

Learning focus: How the web works—browsers, servers, URLs, HTTP/HTTPS—and the difference between client-side and server-side code.

3.1 The World Wide Web (WWW)

Web browsers (Chrome, Firefox, Edge, Safari) request pages from web servers, receive HTML/CSS/JS, and display them. A URL (e.g. https://example.com/page) identifies the resource. HTTP is the protocol for sending requests and responses; HTTPS is HTTP with encryption (SSL/TLS) for security.

Example: URL parts
https://www.school.edu.lk/courses — https = protocol, www.school.edu.lk = domain, /courses = path.
  • Web Browsers – display content, run JavaScript
  • Web Servers – store and send files, run server code
  • URLs – address of a resource
  • HTTP & HTTPS – request/response protocol (HTTPS = secure)

3.2 Client-side vs Server-side Scripting

Client-side code runs in the user’s browser (HTML structure, CSS style, JavaScript behaviour). Server-side code runs on the server (e.g. PHP, Node.js) and can access databases and generate HTML before sending it to the browser.
Type Runs on Examples
Client-side User’s browser HTML, CSS, JavaScript
Server-side Web server PHP, Node.js, Python
  • Client-side (HTML, CSS, JavaScript)
  • Server-side (PHP, Node.js)

04 Hypertext Markup Language (HTML) (15 Hours)

Learning focus: Structure web pages with HTML—elements, tags, links, images, lists, tables, forms—and use semantic HTML for accessibility.

4.1 Introduction to HTML

HTML (HyperText Markup Language) is the standard language for creating web pages. Browsers read HTML tags and display content. HTML has evolved from early versions (HTML 2, 3.2) to HTML5, which added semantic elements and better support for media and forms.

HTML history: From simple document markup to HTML5 (current standard), supporting video, audio, canvas, and semantic structure.

  • What is HTML?
  • HTML history (HTML5 and standards)

4.2 HTML Elements & Structure

Elements use tags (e.g. <p>, <h1>) and attributes (e.g. href, src, id). The head holds meta information, title, and links to CSS/JS; the body holds the visible content. Use headings (h1–h6) for hierarchy and paragraphs (<p>) for text blocks.

Tags come in pairs: opening <tag> and closing </tag>. Some elements are void (no closing tag): <img>, <br>, <input>. Attributes go inside the opening tag: name="value". Common ones: id (unique), class (reusable), href (link URL), src (image/script source).

Example: minimal HTML5 page structure and semantic layout.

HTML – Basic page structure (index.html)
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>My Page</title>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <header>
    <h1>Welcome</h1>
    <nav>
      <a href="index.html">Home</a> |
      <a href="contact.html">Contact</a>
    </nav>
  </header>
  <main>
    <p>This is the main content area.</p>
  </main>
  <footer>&copy; 2025</footer>
  <script src="script.js"></script>
</body>
</html>

Links: <a href="url">text</a>. Images: <img src="path" alt="description">—always use alt for accessibility. Lists: <ul><li> (unordered) or <ol><li> (ordered). Tables: <table>, <tr>, <td>, <th>. Forms: <form action="..." method="get|post"> with inputs (text, email, password, submit, etc.), labels, and optional validation (required, type).

Semantic elements describe meaning: <header>, <footer>, <section>, <article>, <nav>. They help screen readers and search engines understand the page. Use alt text on images; keep a logical heading order (h1 → h2 → h3); associate labels with form controls.

05 Cascading Style Sheets (CSS) (12 Hours)

Learning focus: Style web pages with CSS—where to put CSS, selectors, colours, fonts, box model, and layout (Flexbox/Grid).

5.1 Introduction to CSS

CSS (Cascading Style Sheets) controls how HTML looks: colours, fonts, spacing, layout. You can add CSS in three ways: inline (style attribute on one element), internal (<style> in head), or external (separate .css file linked in head). External is best for maintainability and re-use across pages.
Inline styles apply to a single element: <p style="color: red;">. Internal CSS goes in <style></style> in the <head> and applies to that page only. External CSS is in a separate file (e.g. styles.css) and is linked with <link rel="stylesheet" href="styles.css"> so many pages can share the same styles.

Examples: inline, internal, and external CSS.

CSS – Inline (on one element) vs Internal (in head) vs External (file)
/* INLINE – in HTML: */
<p style="color: blue; font-size: 18px;">Text</p>

/* INTERNAL – inside <style> in <head>: */
p { color: blue; font-size: 18px; }

/* EXTERNAL – in styles.css, linked by: */
<link rel="stylesheet" href="styles.css">
  • Inline, Internal, External CSS

5.2 CSS Selectors & Properties

Selectors target elements: by element (e.g. p { }), by class (e.g. .highlight { }), by ID (e.g. #main { }). You can combine: div.box (div with class box), ul li (li inside ul). Common properties: color, font-family, font-size; background-color, background-image.

Colours can be: name (red), hex (#ff0000 or #f00), rgb(255, 0, 0), or rgba(255, 0, 0, 0.5) for transparency. Fonts: use font-family (e.g. Arial, sans-serif), font-size (e.g. 1rem, 16px), font-weight (bold/normal), line-height for readability.

Example: selectors, colours, fonts, box model, and Flexbox.

CSS – Selectors, box model, Flexbox (styles.css)
/* Element and class selectors */
body { font-family: Arial, sans-serif; color: #333; }
.highlight { background-color: #fef3c7; padding: 4px 8px; }
#main { max-width: 900px; margin: 0 auto; }

/* Box model */
.card {
  padding: 1rem;
  border: 1px solid #e2e8f0;
  margin: 1rem 0;
  border-radius: 8px;
}

/* Flexbox row */
.row {
  display: flex;
  gap: 1rem;
  flex-wrap: wrap;
}

/* Responsive: stack on small screens */
@media (max-width: 600px) {
  .row { flex-direction: column; }
}

Example: CSS Grid for a 2D layout (columns and rows).

CSS – Grid layout (columns, rows, gap)
/* Grid: 3 equal columns, gap between items */
.grid-container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;   /* or repeat(3, 1fr) */
  gap: 1rem;
}

/* Responsive: 2 cols on tablet, 1 on mobile */
@media (max-width: 768px) {
  .grid-container { grid-template-columns: 1fr 1fr; }
}
@media (max-width: 480px) {
  .grid-container { grid-template-columns: 1fr; }
}
  • Element, Class, ID selectors (and combinators: space, .class)
  • Colors (hex, rgb, names), Fonts, Backgrounds
  • Grid: grid-template-columns, gap, repeat()

5.3 Styling Web Pages

The Box Model: each element has content, padding, border, and margin. Flexbox (display: flex) arranges items in a row or column with alignment and spacing. Grid (display: grid) creates two-dimensional layouts with rows and columns. Media Queries (@media (max-width: 768px) { }) apply different styles for different screen sizes—essential for responsive design.

  • Box Model
  • Flexbox
  • Grid
  • Responsive design (Media Queries)

06 Introduction to Front-End Development Tools (03 Hours)

Learning focus: When and why to use JavaScript, TypeScript, and Next.js in front-end development.

6.1 JavaScript

JavaScript (JS) runs in the browser and adds behaviour to web pages: form validation, menus, animations, fetching data without reloading the page. Use it when you need interactivity, dynamic content, or to talk to APIs.

Purpose: Interactivity, DOM manipulation, form checks, AJAX/fetch. When to use: Any site that needs buttons, dropdowns, sliders, or live updates.

6.2 TypeScript

TypeScript is JavaScript with static types (you declare types for variables and function parameters). It helps catch errors in the editor and scales better in large projects. Use it when building bigger apps or when working in a team.

6.3 Next.js

Next.js is a React-based framework for building full websites: server-side rendering (SSR), static generation, file-based routing, and API routes. Use it when you need SEO-friendly pages, fast loading, or a full-stack React app.

07 JavaScript Fundamentals (06 Hours)

Learning focus: Core JavaScript: variables, data types, operators, functions, and control flow (if, loops).

7.1 Introduction to JavaScript

JavaScript can be added in HTML with <script> (inline or external file). It runs in the browser and can change the page (DOM), react to events (click, submit), and communicate with servers.

Where to put <script>: Placing the script at the end of body (before </body>) ensures the HTML is loaded before the script runs. For external JS: <script src="script.js"></script>. To run code only after the page is ready, use document.addEventListener('DOMContentLoaded', function() { ... });.

7.2 Variables & Data Types

Declare variables with let (changeable) or const (constant). Data types: numbers, strings (text in quotes), booleans (true/false), arrays [], objects {}, and undefined/null.

Example: variables
let name = "Ali"; const score = 85; let passed = true;

Example: arrays – create, length, access by index, loop.

JavaScript – Arrays (create, index, length, for loop)
const fruits = ["Apple", "Banana", "Orange"];
const len = fruits.length;        // 3
const first = fruits[0];         // "Apple" (index starts at 0)
fruits.push("Mango");             // add at end

// Loop over array
for (let i = 0; i < fruits.length; i++) {
  console.log(fruits[i]);
}
// Or: for (const item of fruits) { console.log(item); }

7.3 Operators

Arithmetic: + - * / %. Comparison: == === != !== < > <= >=. Logical: && (and), || (or), ! (not). Use === for equality to avoid type coercion. String concatenation: "Hello " + name or template literal `Hello ${name}`.

7.4 Functions

Functions bundle reusable code. Define with function name() { } or const name = () => { }. They can take parameters and return a value. return sends a value back; without it the function returns undefined.

7.5 Control Structures (if, loops)

if / else if / else for decisions. for and while for repetition. for (let i = 0; i < arr.length; i++) or for (const item of arr) to loop over arrays.

Example: variables, function, if/else, and button click handler.

JavaScript – Variables, function, condition, event (script.js)

Web hosting is a service that stores your website files on a server connected to the internet so that visitors can access your site 24/7. The host provides space, bandwidth, and often email and databases.

10 Domain Names and DNS (03 Hours)

Learning focus: What a domain is, how to register one, how DNS works, and how to configure DNS for your site.

10.1 What is a Domain Name?

A domain name is the human-friendly address of your website (e.g. myschool.edu.lk). It maps to an IP address (e.g. 192.168.1.1) so users can type a name instead of numbers. It is your online identity.

10.2 Domain Registration

You register a domain through a registrar (e.g. GoDaddy, Namecheap, or local registrars for .lk). You pay yearly. Choose a name that is short, memorable, and matches your brand. Check availability first.

10.3 Understanding DNS

DNS (Domain Name System) translates domain names to IP addresses. When someone types your domain, their browser asks DNS servers “What IP is this?” and gets the server address. Records: A (domain → IP), CNAME (alias), MX (mail), etc.

Example: common records
A – example.com → 203.0.113.5
CNAME – www.example.com → example.com
MX – mail server for email

10.4 Configuring DNS Settings

At your registrar or host you edit DNS settings: add an A record pointing your domain to your hosting server’s IP, and often a CNAME for www. Changes can take a few minutes to 48 hours to spread (DNS propagation).

11 Deploying a Website (03 Hours)

Learning focus: Upload files (FTP or File Manager), set permissions, backups, and basic database setup.

11.1 Uploading Website Files

To put your site live, you upload your HTML, CSS, JS, and other files to the hosting server. The main folder is often public_html or www. Use FTP (File Transfer Protocol) with a client like FileZilla, or the host’s File Manager in the browser.

FTP: Connect with host, username, password, and port (often 21). Upload to the correct folder; keep the same folder structure as on your computer. File Manager: Log into cPanel (or similar), open File Manager, go to public_html, and upload or create files there.

  • FTP (client: FileZilla, etc.)
  • File Manager (cPanel)

11.2 Basic Server Administration

File permissions: Typically folders 755, files 644. Restrict sensitive files (e.g. config) so they are not readable by the public. Backups: Use cPanel backup or a plugin; keep regular copies of files and database off the server. Database setup: Create database and user in cPanel; note host, database name, username, password; use these in your app (e.g. WordPress, custom script).

  • File permissions (755, 644)
  • Backups

Practical Chapters

P1 Practical 01 – Website with HTML, CSS & JavaScript

Objective: Build a small multi-page or single-page website using only HTML, CSS, and JavaScript, with clear navigation and simple interactions.

Requirements

  • HTML – Valid structure, semantic tags (header, nav, main, footer), links, images (with alt), lists or tables where suitable.
  • CSS – External stylesheet; layout (e.g. Flexbox or Grid), colours, fonts, and at least one media query for different screen sizes.
  • JavaScript – At least one interactive feature (e.g. button click, form validation, show/hide section, simple menu toggle).
  • Navigation & interactions – Easy-to-use navigation between pages or sections; interactions should work and be clearly visible.

Suggested steps

  1. Plan: 2–3 pages or sections (e.g. Home, About, Contact) and one or two interactions.
  2. Create the HTML structure for each page/section; add content and links.
  3. Create a single CSS file; style layout, typography, and make it responsive.
  4. Add JavaScript for your chosen interaction(s); test in the browser.
  5. Check links, images, and responsiveness on a small screen.
Check your work
  1. Does every page have a clear title and working navigation?
  2. Do all images have alt text? Does the layout work on a narrow screen?
  3. Can you describe what your JavaScript does and where it is used?

P2 Practical 02 – Website using Web Authoring Tool or CMS

Objective: Create a website using a web authoring tool (e.g. Adobe Dreamweaver) or a Content Management System (e.g. WordPress, Joomla).

Requirements

  • Web Authoring Tool – Use a tool such as Dreamweaver to design and build a multi-page site (templates, links, images, basic styling).
  • OR CMS – Install and use WordPress/Joomla (or similar): choose a theme, create pages, add content, menus, and at least one plugin or extension if required.

Suggested steps

  1. Choose: authoring tool or CMS; install or open the software.
  2. Plan: site structure (e.g. Home, Services, About, Contact) and main content.
  3. Create pages and add text, images, and links; set up navigation/menus.
  4. Apply a consistent look (theme or template) and adjust if needed.
  5. Preview and test links and layout on different devices if possible.
Check your work
  1. Can you explain the advantage of using a CMS or authoring tool over hand-coding for this site?
  2. Is the navigation clear and do all links work?

P3 Practical 03 – Host and Deploy Website

Objective: Host the website you created (in P1 or P2), configure the domain (or subdomain), and deploy it so it is accessible on the internet.

Requirements

  • Host the website – Use a hosting account (shared or free tier). Upload files via FTP or File Manager to the correct folder (e.g. public_html).
  • Configure domain – Point a domain (or subdomain) to your hosting using DNS (A record or CNAME as instructed by the host).
  • Deploy live – Ensure the site loads at the given URL; fix any path or permission issues. Optionally add SSL (HTTPS) if available.

Suggested steps

  1. Get hosting and note: FTP/cPanel login, public_html path, and IP or nameservers.
  2. Upload your site files (or export/import if using CMS) and set file permissions if needed.
  3. In DNS, add A record (or CNAME) so your domain points to the host; wait for propagation.
  4. Open the site in a browser; test homepage, links, and images. Fix broken paths if any.
  5. Enable HTTPS in cPanel/host if offered; confirm the site works over https.
Check your work
  1. Can someone else open your site using the URL you configured?
  2. What DNS record did you use to point the domain to your host?
  3. What would you do next to keep the site secure and backed up?
