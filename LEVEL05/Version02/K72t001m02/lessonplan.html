System Analysis and Design

Module Code: K72T001M02 / Level 05  |  Module Type: Compulsory  |  Relevant Unit/s: K72T001U02  |  Pre-requisite: K72T001M01

Module Aim

To enable the students to apply concepts related to identifying, analyzing, prioritizing, documenting, and managing user requirements for software solutions by applying requirement gathering and requirement identification techniques.

General Objective

At the end of this module, students will be able to apply system analysis and design concepts to analyze requirements from documentation, produce logical designs using appropriate techniques, identify and document system components and data, document requirement outcomes using standard templates, develop test cases from design specifications, and prepare a complete requirement document.

Specific Objectives

At the end of this module the student shall be able to:

  1. Analyze requirements specified in documentation and extract key information for design and implementation.
  2. Apply system analysis techniques to produce a logical design for a given software solution.
  3. Identify system components and document functionality and data flows using appropriate notations.
  4. Document requirement outcomes according to standard templates and conventions.
  5. Develop test cases based on design specifications to verify and validate the solution.
  6. Prepare the final requirement document incorporating analysis, design, and test specifications.
